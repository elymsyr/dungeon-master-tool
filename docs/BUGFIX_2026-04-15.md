# Bug Fixes — 2026-04-15

## 1. Mind map resize hitbox (mobile + desktop)
Corner resize handles had a 10×10 px hit area — impossible to grab with a finger, fiddly even with a mouse.
**Fix:** hit area bumped to **44×44 on touch**, **24×24 on desktop**. Visual square still 10×10, centered.
- `lib/presentation/screens/mind_map/mind_map_node_widget.dart` — `_buildCornerHandle`

## 2. macOS login failure (root cause)
Sign-in on macOS failed outright with a socket/URL error. Cause: App Sandbox was on but `com.apple.security.network.client` was missing — all outbound sockets blocked.
**Fix:** added `network.client` entitlement to both profiles; Release entitlements file also got the sandbox entry it was missing.
- `macos/Runner/DebugProfile.entitlements`
- `macos/Runner/Release.entitlements`

Bonus: sign-in errors now `debugPrint` through `LogBuffer` so future auth failures show up in the bug report dialog.
- `lib/application/providers/auth_provider.dart` — `signIn`

## 3. Offline startup "ClientException" leak
Launching the app with no internet surfaced raw `ClientException` / `SocketException` text in the UI. Didn't crash, but ugly.
**Fix:**
- New util `lib/core/utils/error_format.dart` — `isOfflineError(e)` + `formatError(e)` detect network failures (SocketException, HandshakeException, TimeoutException, HttpException, http.ClientException, common message substrings) and return a friendly "You're offline" string.
- `currentProfileProvider` / `profileByIdProvider` now swallow offline errors and return `null` (no AsyncError state at all).
- All `Error: $e` display sites in social (marketplace / feed / players / messages), profile, and the follow/listing dialogs now route through `formatError`.

## 4. Cloud backup & marketplace offline messages
Cloud backup upload/restore, storage info, backup list, marketplace panel, and my-snapshots panel all showed raw exception strings when the device was offline.
**Fix:** `CloudBackupOperationState.error(...)` wraps every error through `formatError` at the source, and each consumer UI (save sync dialog, marketplace panel, my snapshots panel) checks `isOfflineError` to swap in a friendly "You're offline — …" message.
- `lib/application/providers/cloud_backup_provider.dart`
- `lib/presentation/widgets/save_sync_indicator.dart`
- `lib/presentation/widgets/marketplace_panel.dart`
- `lib/presentation/widgets/my_snapshots_panel.dart`
