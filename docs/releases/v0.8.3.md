# Release Notes — alpha-v0.8.3

**Release Date:** 2026-03-31
**Tag:** `alpha-v0.8.3`
**Status:** Alpha

---

## Overview

v0.8.3 focuses on two pillars: **architecture quality** and **UX polish**. The most visible change is the new global edit mode that locks the entire app into a read-only state by default — a significant quality-of-life improvement for DM sessions where accidental edits were a persistent risk. Under the hood, a pub-sub EventBus replaces several fragile direct signal chains as part of the ongoing Phase 4 architecture work. Soundpad crossfades and state-button highlighting were also corrected in this release.

---

## What's New

### Global Edit Mode

A single **✏️ button** in the main toolbar now acts as a master read/write toggle for the entire application.

- **Edit mode OFF (default):** all text fields across the app are read-only — NpcSheet cards, session log, DM notes, mind map note nodes and entity nodes. No accidental edits during a live session.
- **Edit mode ON:** all fields unlock simultaneously. Turning edit mode back off auto-saves any dirty NpcSheet cards before locking.
- New sheets opened while edit mode is active inherit the current state immediately.
- The toolbar button highlights using each theme's accent colour when active.

**Affected areas:** Database tab (all open NpcSheet cards), Session tab (event log, DM notes, quick log entry), Mind Map (note nodes, embedded entity nodes).

### EventBus — Phase 4 Architecture

Introduced `core/event_bus.py`, a lightweight publish-subscribe event bus for cross-cutting application events.

- **Events defined:** `entity.deleted`, `entity.created`, `entity.updated`, `theme.changed`, `language.changed`, `edit_mode.changed`
- Handler exceptions are caught and logged without interrupting other subscribers.
- Duplicate subscriptions are silently ignored.
- Replaces the fragile `db_tab.entity_deleted → entity_sidebar.refresh_list` direct signal chain. The sidebar now reacts to all three entity lifecycle events automatically.
- `change_theme()` and `change_language()` publish their respective events so future subscribers can react without manual wiring in `main_root.py`.

### Soundpad: Crossfade & State Button Fixes

- **Fixed abrupt music-state transitions.** Root cause: the `fade_ratio` QPropertyAnimation was triggering a 1500ms per-player fade on every frame (~16ms), effectively preventing any volume change. Player volumes are now set directly during crossfade via `apply_volume_direct()`. The per-player `fade_to()` is reserved for intensity-layer transitions only.
- Crossfade duration increased from 2000ms → **3000ms**; easing curve changed to **InOutCubic** for a smooth S-curve blend.
- **Fixed active state button not highlighting** on initial theme load. `SoundpadPanel` now listens to the `state_changed` signal and syncs all buttons on every state change, not just on user click.

## Internal / Developer Changes

- `core/event_bus.py` — new file, ~55 LOC
- `MainWindow` gains `global_edit_mode: bool` and `event_bus: EventBus` instance
- `DatabaseTab`, `SessionTab`, `MindMapTab`, `EntitySidebar` all accept an optional `event_bus` constructor argument
- `toggle_active_edit_mode()` is now a true broadcast instead of a focused-widget toggle
- All 11 QSS theme files updated with `QPushButton#editModeBtn` styles

---

## Upgrade Notes

- No data migration required. Existing world files (`.dat`) are fully compatible.
- The app now starts in **read-only mode**. Press ✏️ or `Ctrl+E` to enable editing — this is intentional.

---

## Known Issues

- Soundpad `mid-length` transition sounds between loops are not yet supported (tracked in TODO.md).
- Mind map note nodes with no content may show edit mode toggle as disabled even when global edit is ON — open the node context menu and add content to activate.
