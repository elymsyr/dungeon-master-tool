# Release Notes — alpha-v0.8.4

**Release Date:** 2026-04-01
**Tag:** `alpha-v0.8.4`
**Status:** Alpha

---

## Overview

v0.8.4 is a major architecture and UI polish release. It completes Phases 2–4 of the IMPROVEMENT_ROADMAP (god class decomposition, UI consistency, architecture patterns) and delivers several PRE_ONLINE milestones including the embedded PDF viewer, right-side PDF panel, spells tab refactoring, and comprehensive button/theme standardization. Over 40 files were touched across 11 themes, 4 locales, and the full UI layer.

---

## What's New

### Architecture — God Class Decomposition (Phase 2)

**NpcSheet** (1497 LOC) decomposed into an orchestrator facade (~530 LOC) + 5 focused sub-widgets:
- `npc_sheet_stats_tab.py` — ability scores, combat stats, defense
- `npc_sheet_actions_tab.py` — traits, actions, reactions, legendary actions
- `npc_sheet_inventory_tab.py` — equipment/items
- `npc_sheet_spells_tab.py` — spells (linked + manual)
- `npc_sheet_helpers.py` — shared `make_section`, `create_feature_card`, `clear_section`

**CombatTracker** (~632 LOC) decomposed into a thin coordinator + 3 sub-widgets:
- `combat_combatant_list.py` — table, row rendering, HP editing
- `combat_controls.py` — encounter selector, turn navigation, quick-add form
- `combat_table.py` — HP bar widget, condition icons, draggable table

### Architecture — API Client Consolidation (Phase 4)

`core/api/` package created with clean source abstraction:
- `client.py` — unified API client
- `base_source.py` — abstract base for API sources
- `dnd5e_source.py` / `open5e_source.py` — concrete implementations
- `field_mappers.py` / `entity_parser.py` — shared parsing logic

### Architecture — MVP/Presenter Layer (Phase 4)

`ui/presenters/` package introduced:
- `combat_presenter.py` — encounter/turn/combatant business logic
- `npc_presenter.py` — entity load/save/discard/delete lifecycle

### Spells Tab Refactoring

- Manual spells section removed entirely — replaced with a **Manual Add dialog**
- Dialog includes all spell fields (Level, School, Casting Time, Range, Duration, Components) + a "Save to database" checkbox
- Custom (inline) spells and database-linked spells now render in a **unified list**
- `LinkedEntityWidget` extended with custom entry API and `manual_add_requested` signal

### Right-Side PDF Panel

- New `ui/pdf_panel.py` — collapsible right-side panel (like the soundpad) for viewing PDFs
- **"Project PDF"** button added to entity card Docs tab — sends selected PDF to the panel
- Soundpad and PDF panel are **mutually exclusive** — opening one closes the other
- Toggle button (file icon) in the main toolbar

### PDF Viewer Improvements

- **Middle-mouse drag** to pan/scroll PDF pages
- Removed "Folder…" button — only "Open…" remains
- Navigation emoji buttons (◀/▶) replaced with Qt standard icons

### Theme & Background Color Fix

- All 11 theme files: `QComboBox` and `QSpinBox` added to `featureCard` transparent rule
- `common.qss`: `QLabel`, `QListWidget`, `:focus` transparency rules for `#sheetContainer`
- Fixes visible background mismatch on Name/Type/Source fields, Challenge Rating, spell contents across all themes

### Button Standardization

- **HP +/- buttons**: inline `setStyleSheet()` removed, themed via QSS objectNames (`hpDecreaseBtn`/`hpIncreaseBtn`) with theme-appropriate red/green colors
- **`compactBtn`** QSS class added for small icon-only buttons (28x28)
- **Emoji → Icon replacement**: ↑/↓/◀/▶/</> buttons across 4 files replaced with Qt standard icons
- **Combat controls**: `btn_quick_add` → successBtn, `btn_add` → successBtn, `btn_add_players` → primaryBtn, `btn_roll` → primaryBtn
- **Edit mode button**: now shows `"✏️ Edit"` text for consistent sizing
- All small icon buttons standardized to 28x28

### Combat Table Improvements

- Init/AC columns: `ResizeToContents` (fit content exactly)
- HP/Conditions columns: `Stretch` (share remaining space equally)
- Init/AC cells center-aligned; Name left-aligned
- HP bar: border-radius removed for flat look matching buttons

### Height/Width Auto-Sizing

- `list_residents`: removed hard `setMaximumHeight(80)`, added auto-fit height
- All containers (`list_battlemaps`, `list_pdfs`, `LinkedEntityWidget`, `make_section`) changed from `Preferred` to `Expanding` size policy
- Horizontal scrollbar disabled on entity lists, word wrap enabled

### Battle Map Toolbar Layout

- DM toolbar split from 1 row into 2 rows:
  - **Row 2**: Tool selector + action buttons (Navigate, Ruler, Circle, Draw, Fog, Fill, Clear)
  - **Row 3**: Grid controls (Grid, Cell Size, Snap, Feet Per Cell)
- Tool buttons no longer clipped in themes with longer translated labels

### Inline CSS Cleanup (Phase 4)

- `themes/common.qss` expanded to 15+ objectName-based QSS rules
- Migrated `setStyleSheet()` calls to `setObjectName()` across 6 files
- HP button colors defined per-theme in all 11 `.qss` files
