# Entity Audit Log тАФ Official & Built-in Packages

> Automated System Architecture Inspector тАФ per-entity ledger for the
> `dungeon-master-tool` D&D 5e content packages. Companion roadmap:
> [`system_mechanics_roadmap.md`](system_mechanics_roadmap.md). Branch: `list`.

## Scope & method

Two package sources were inspected, both shipped as the runtime's official /
built-in content:

1. **Official first-party catalog тАФ Open5e packs** (machine-imported, JSON
   assets): `flutter_app/assets/open5e_packs/*.pkg.json`, listed in
   `manifest.json` + `first_party/manifest.json`. **19 packs ┬╖ 20,712 entity
   cards.** Every card was parsed from JSON; its `attributes` map was compared
   field-by-field against the typed schema (`builtin/content.dart`) and the
   mechanics the runtime actually consumes (`character_resolver.dart`,
   `pending_choice_resolver_dialog.dart`, the chargen resolvers).
2. **Built-in pack тАФ SRD 5.2.1 core** (hand-authored Dart):
   `flutter_app/lib/domain/entities/schema/builtin/srd_core/`. **~2,260 cards.**
   These are reviewed at the per-builder / per-category level (the rows are
   generated by typed Dart builders, so deficiencies are uniform within a
   builder); representative named rows are cited from source.

Because the chargen entities (feats / backgrounds / classes / subclasses /
species / subspecies тАФ 270 official cards) are where the three inspection
criteria bite hardest and vary card-to-card, **they are enumerated
individually** (per pack, below). The bulk content (spells, magic items, gear,
traits, creature-actions, monsters тАФ 20,442 official cards) is **machine-uniform
within each pack├Чcategory**, so it is audited at that granularity with exact
counts and representative named rows тАФ a name-by-name transcription of 20k
identical-shaped rows would add length without information.

### Verdict legend
- **Clean** тАФ every described prerequisite sits in a typed, *enforced* field;
  every described mechanic has a matching typed `effects`/attribute the runtime
  reads; nothing material is stranded in a generic text field.
- **Unimplemented Prerequisite** тАФ the card's text states a requirement that no
  typed/enforced field carries (so the gate never fires).
- **Missing Mechanic** тАФ a functional rule in the card's text has no
  corresponding typed `effects`/attribute the resolver can apply.
- **Poor Data Structure** тАФ content dumped into a generic `description` /
  `effects`-as-prose field instead of dedicated typed fields.

### B1 ┬╖ Spells тАФ 1,297 cards (9 packs) тАФ **Missing Mechanic + Poor Data Structure**

**Typed today:** `level`, `school_ref`, `casting_time_amount/unit_ref`,
`range_type`/`range_ft`, `components`, `duration_amount/unit_ref`,
`requires_concentration`, `is_ritual`, `material_description/cost_gp/consumed`,
and тАФ partially тАФ `save_ability_ref` (567/1,297) and `damage_type_refs`
(294/1,297) and `attack_type` (89/1,297).

**Verdict тАФ every spell card:** *Missing Mechanic.* There is **no `damage_dice`
/ amount field, no per-slot or per-level scaling, no condition-application, and
no `effects` channel of any kind.** The save/damage-type tags that exist are
inert classification labels тАФ nothing rolls damage, applies a save outcome, or
scales the spell. 100% of each spell's actual behavior lives in the prose
`description`. The 730 spells lacking even the `save_ability_ref`/
`damage_type_refs` tags are additionally *Poor Data Structure* (the save/damage
that the text clearly describes is not even classified).
Representative: *Fireball*, *Eyebite*, *Cure Wounds*, *Counterspell* тАФ all
identity-typed, all behavior in prose.

### B2 ┬╖ Magic items тАФ 1,063 cards (Vault of Magic) тАФ **Missing Mechanic + Poor Data Structure**

**Typed today:** `rarity_ref`, `magic_category_ref`, `activation`,
`requires_attunement` (boolean; true for 452/1,063), `is_cursed`, `is_sentient`,
`weight_lb` (114/1,063).

**Verdict тАФ every magic-item card:** *Poor Data Structure* **and** *Missing
Mechanic.* The `effects` attribute is populated for all 1,063 items but in
**every case is a verbatim copy of the prose `description`** тАФ it is not a typed
effect list, so no bonus, resistance, attack rider, or charge/recharge is
applied by the resolver. **Unimplemented Prerequisite:** where an item's text
restricts attunement ("requires attunement by a spellcaster / by a wizard"), the
class/spellcasting restriction is left in prose тАФ the typed `attunement_prereq`
field (schema `content.dart:1002`) is never populated, so the restriction is
unenforced. `requires_attunement` carries only the yes/no, not the *by whom*.
Representative: *Aberrant Agreement*, *Staff of Power*, *Robe of the Archmagi*.

### B3 ┬╖ Adventuring gear тАФ 159 cards (5 packs) тАФ **Poor Data Structure (empty cards)**

**Verdict тАФ every gear card:** *Poor Data Structure.* These are the worst-typed
cards in the catalog: **every one ships with `cost_cp: 0`, `weight_lb: 0.0`,
empty `description`, `consumable: false`, `is_focus: false`** тАФ i.e. name +
category only, with no economic, encumbrance, focus, or rules data at all. The
importer mapped no cost/weight/text for the gear category. Representative (all
identical shape): *Holy Symbol*, *Thieves' Tools*, *Disguise Kit*, *Forgery
Kit*, *Common Clothes*. (Contrast: built-in SRD gear is fully cost/weight typed.)

### B4 ┬╖ Traits тАФ 6,423 cards (6 monster packs) тАФ **Missing Mechanic**

**Typed today:** `trait_kind`, `source`, `description`.

**Verdict тАФ every trait card:** *Missing Mechanic.* The trait shape has **no
effects channel** тАФ every monster/species special ability (Pack Tactics, Magic
Resistance, Regeneration, Legendary Resistance, innate-spellcasting trait lines,
aura/save-or-suck riders) is a prose `description` string with no typed effect,
DC, recharge, or damage. None are mechanically applied. This is the single
largest cohort of unenforced rules in the catalog.

### B5 ┬╖ Creature-actions тАФ 8,615 cards (6 packs) тАФ **mixed: ~41% typed, ~59% prose**

**Typed today:** `action_type`, `is_attack`, `attack_kind`, `attack_bonus`,
`reach_ft`, `range_normal_ft`/`range_long_ft`, `damage_dice`, `recharge_kind`/
`recharge_min_roll`, `uses_per_day`.

**Verdict тАФ attack actions (~3,549): mostly Clean** for the to-hit + damage line
(attack bonus, reach/range, damage dice, recharge are all typed). Residual rider
text ("and the target is grappled, escape DC 13", extra on-hit damage, secondary
saves) remains in `description` тЖТ *Missing Mechanic* for the rider only.
**Verdict тАФ non-attack actions (~5,066): Missing Mechanic.** Breath weapons,
save-or-suck actions, summons, recharge AoEs carry `recharge_kind`/`uses_per_day`
but **no typed save DC/ability or effect** тАФ the whole behavior is prose.

### B6 ┬╖ Monsters тАФ 2,885 cards (6 packs) тАФ **mostly Clean (identity), Missing Mechanic (folded abilities)**

**Typed today:** a structured `stat_block` map plus `ac`, `hp_average`/`hp_dice`,
`speed_*_ft`, `cr`, `xp`, `proficiency_bonus`, `passive_perception`, `size_ref`,
`creature_type_ref`, `alignment_ref`, `language_refs`, `resistance_refs`,
`action_refs`, `trait_refs`. Defensive immunities/resistances are typed.

**Verdict тАФ every monster card:** *Clean for the stat block* (the numeric combat
identity is fully typed and not prose-dumped). The *Missing Mechanic* it inherits
is entirely downstream: its referenced `trait_refs` (B4) and non-attack
`action_refs` (B5) are unenforced prose, so the monster displays correctly but
its special abilities don't execute. No monster is *Poor Data Structure*.

## Part C тАФ Official catalog: chargen entities (per-entity ledger)

Enumerated individually. For these categories the runtime *does* consume typed
fields (`CharacterResolver`, `level_up_planner`, the feat-prereq gate in
`pending_choice_resolver_dialog._computeEligibleFeats`), so the verdict reflects
whether each card's text is actually wired.

> Recurring structural notes (apply to every card of the type unless "Clean"):
> - **subclass / class** тАФ the chargen schema has **no typed leveled-feature
>   model** (no per-level feature rows, no `at_level` on imported feature
>   prose); only L1 identity fields are typed, so all L1тАУ20 progression is prose.
> - **background** тАФ grants (skills/languages/ASI/equipment) are typed, but the
>   background *feature* is always prose (no feature/effects channel).
> - **feat** тАФ the prereq gate *is* enforced when `prereq_clauses` (or the legacy
>   `prereq_ability_ref`/`prereq_min_character_level`) are populated; a feat with
>   only the freeform `prerequisite` string is **not** gated.

## Part D тАФ Built-in SRD 5.2.1 core (per-category audit)

Hand-authored in `builtin/srd_core/`. Unlike the imported catalog, the
class/subclass/feat layer here is **mechanically wired** via the inline effect
DSL; the spell / magic-item / trait layer is **not**.

### D1 ┬╖ Player feats тАФ `feats.dart` (~62) тАФ **mostly Clean**
Typed `category_ref`, `repeatable`, ASI gate (`asi_ability_options/amount/
max_score`), typed prereqs (`prereq_min_character_level`,
`prereq_requires_spellcasting`, `prereq_ability_ref`+`prereq_min_score`), and an
inline `effects` list (18 effect blocks: `choice_group`, ability/skill grants).
Most are **Clean** (prereq enforced + benefit typed). Cards whose benefit is a
pure narrative rider keep a `benefits` markdown body тЖТ *Missing Mechanic* for
that rider (e.g. *Lucky*, *Tough* are typed; narrative-only boons are not).

### D2 ┬╖ Class / subclass feature feats тАФ `feats_class.dart` (~236) тАФ **mostly Clean**
Auto-granted features (`auto_granted_by` + `at_level` gates) carrying 108
`effect(...)` calls + 18 inline `effects` lists. These are the wiring that makes
built-in classes/subclasses mechanical (Rage, Sneak Attack, Extra Attack,
Metamagic, Eldritch Invocations, etc.). **Clean** where the feature maps to an
effect kind; **Missing Mechanic** for features the resolver has no kind for
(narrative/utility features kept as prose).

### D3 ┬╖ Classes тАФ `classes.dart` (12) & Subclasses тАФ `subclasses.dart` (12) тАФ **Missing Mechanic (leveled prose)**
L1 identity typed (hit die, saves, proficiencies, caster kind, skill choices);
L1 features auto-grant the D2 feature-feats (so they *are* wired). Higher-level
progression that is not expressed as an auto-granted feature remains prose тЖТ
*Missing Mechanic* for the non-feat-backed portions; no per-level feature row.

### D4 ┬╖ Species тАФ `species.dart` (9) & Subspecies тАФ `subspecies.dart` (22) тАФ **partial: typed grants + prose traits**
Size/speed/senses/ASI/damage-resistance typed; but each species' named special
abilities are `trait_refs` into `traits.dart` (D7) тЖТ *Missing Mechanic* (the
trait itself is prose). *Poor Data Structure* does not apply (grants are typed).

### D5 ┬╖ Backgrounds тАФ `backgrounds.dart` (16) тАФ **Missing Mechanic (feature prose)**
Skill/tool/language/equipment grants typed; the background **feature** is a prose
`benefits`/`description` body with no effects channel.

### D6 ┬╖ Spells тАФ `spells.dart` (341) тАФ **Missing Mechanic + Poor Data Structure**
Zero inline `effects` (grep: 0). Identity typed; all behavior in a `benefits`
markdown body. Same deficiency as official B1 тАФ no dice/save-outcome/scaling.

### D7 ┬╖ Traits тАФ `traits.dart` (238) тАФ **Missing Mechanic**
Builder `_t()` exposes only `name` / `trait_kind` / `description` тАФ **no effects
channel**. Every shared monster/species trait (Pack Tactics, Legendary
Resistance, Undead Fortitude, Spellcasting (Lich), Magic ResistanceтАж) is prose.

### D8 ┬╖ Magic items тАФ `magic_items.dart` (286) тАФ **Missing Mechanic + Poor Data Structure**
Identity typed (`rarity_ref`, `requires_attunement`, etc.); 1 inline `effects`
key across the whole file. Item behavior is prose; attunement-by-class restriction
not typed (`attunement_prereq` unused).

### D9 ┬╖ Creature-actions тАФ `creature_actions.dart` (528) тАФ **mostly Clean (attacks)**
Builder `_a()` types `attack_kind`/`attack_bonus`/`reach`/`range`/`damage_dice`/
`damage_type`/`save_dc`/`save_ability`/`recharge`/`conditions`. Attack & save
actions are well-typed; pure-narrative actions keep prose riders.

### D10 ┬╖ Monsters тАФ `monsters.dart` (248) & animals/mounts/vehicles (~113) тАФ **Clean (stat block)**
Fully typed numeric stat blocks referencing D7 traits + D9 actions. Special
abilities inherit D7's *Missing Mechanic*; the monster row itself is Clean.

### D11 ┬╖ Weapons (39) / Armor (14) / Tools (39) / Ammunition / Packs (7) тАФ **Clean**
Fully typed (`damage_dice`, `damage_type_ref`, `property_refs`, `mastery_ref`,
`cost_gp`, `weight_lb`, range, versatile dice; armor AC; tool cost/weight; packs
reference contained gear). No prose dumping. **Clean.**

## Summary of verdicts

| Verdict | Where it dominates |
|---|---|
| **Clean** | Built-in weapons/armor/tools/packs; built-in player & feature feats; built-in monster stat blocks; official monster stat blocks; official attack creature-actions; well-modeled species/subspecies grants |
| **Missing Mechanic** | **All** spells (1,638), **all** traits (6,661), magic items (1,349), non-attack creature-actions (~5,066), subclass/class leveled features, background features |
