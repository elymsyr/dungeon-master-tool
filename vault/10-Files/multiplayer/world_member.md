# `world_member.dart`

> [!abstract] Primary Purpose
> Freezed/JSON value object mirroring one `public.world_members` row — a single user's membership in an online (shared) world, including the role they hold and optional display fields joined in from `public.profiles`.

## Inputs / Outputs
**Inputs**
- Providers watched / constructor deps: none (plain immutable data class)
- Reads (DAOs / Drift tables): n/a (domain entity)
- Supabase / CDC subscribed: n/a
- Events consumed (EventBus / ChangeBus): n/a
- Triggers: n/a

**Outputs**
- Providers / public API exposed: `WorldMember` immutable class + `WorldMember.fromJson` / `toJson` (via freezed `.g.dart`)
- Writes (Drift tables): n/a
- Supabase pushed / RPC called: n/a
- Events emitted: n/a

## Dependencies & Links
- Depends on: [[world_role]]
- Used by: [[supabase_world_membership_service]], [[world_membership_service]], [[world_membership_provider]], [[world_members_dao]]
- Domain map: [[Multiplayer-and-Online]]

## Key Logic / Variables
- Fields: `worldId`, `userId`, `role` (`WorldRole`), `joinedAt` (DateTime); plus nullable display fields `username`, `displayName`, `avatarUrl`.
- Invariant/gotcha: display fields are NOT columns on `world_members`. There is no direct FK from `world_members` to `profiles` (both reference `auth.users`), so the display fields are populated client-side by a separate `profiles` query and merge (see [[supabase_world_membership_service]] `listMembers` and the `_ProfileBatchLoader` in [[world_membership_provider]]). Raw CDC rows arrive without them.

## Notes
- The Drift-side row type (`WorldMember` generated by Drift from the `WorldMembers` table) is distinct from this domain class despite the name collision — `world_members_dao.dart` returns the Drift row type.
