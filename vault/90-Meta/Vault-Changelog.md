# Vault Changelog

> [!note] Append-only log of vault structural changes. Newest first.

- **2026-06-10** — Rules-engine PR-R5+R6 (constrained choices + level-up rules). New note [[choice_spec]] (pick-N/distribution descriptor; `choice_group` wire = alias). [[rule_compiler]] += `compileChoiceSpecs` + `trigger_args.at_level`/`gate` parsing on rule_effects rows; [[character_resolver]] gates explicit when_level_up rows by level and warn-validates stored `background_asi` against declared distributions; [[pending_choices]] `seedFeatChoicePendings` routes through ChoiceSpec; [[level_up_planner]] consumes when_level_up `choice_spec` rows → featureOptionPicks (hardcoded `featureOptionTriggers`/`_cumulativePickProgression` stay as fallback); catalog += `choice_spec` kind.
- **2026-06-10** — Rules-engine PR-R4 (attunement runtime). Inventory rows gain additive `attuned` key (`{id, equipped, attuned?}`); attune toggle (link icon) on requires-attunement items in the relation-list widget; `RuleConfig.attunementSlots` (default 3); [[rule_compiler]] `compileAttunementPrereq` (typed `attunement_*` fields → prereq_to_attune clauses); [[character_resolver]] folds `when_attuned` rules for attuned rows, slot-cap + restriction gates warn-keep into `unmetPrerequisites`/`warnings`; `EffectiveCharacter` += `attunedItemIds`/`attunementSlotsUsed`/`attunementSlotsMax`; attunement chip on `resolved_grants_card`; stale `body_slot_ref` per-slot-cap comment fixed.
- **2026-06-10** — Rules-engine PR-R3 (editor UI). New note [[derived_rules_panel]] (read-only compiled-rules view on entity cards, wired into `entity_card`). `_FeatEffectRow` gains Trigger dropdown (+`at_level` arg), clause sub-editor for `kind: prerequisite` rows, and a fixed⇄`$field` value toggle; [[rule_compiler]] resolves `{"$field": key}` against the host card (unresolvable → row skipped); `rule_validator` checks trigger placement + `$field` key existence.
- **2026-06-10** — Rules-engine PR-R2 (trigger model + compiler + interpreter). New notes [[rule_trigger]], [[bound_rule]], [[rule_compiler]] + system note [[Rules-Engine-Triggers]]. [[character_resolver]] passes 2/3/4/5/5b/8 now consume compiled BoundRules via `applyBound`; frozen `character_resolver_legacy.dart` + debug parity assert at `effectiveCharacterProvider` (deleted in R7). Catalog gains `prerequisite` kind + `RuleDefinition.allowedTriggers`; validator checks trigger/clauses placement; `featEffectList` wire gains optional `trigger`/`trigger_args`/`clauses` keys.

[[character_resolver]] gains Pass 10 prerequisite validation + `EffectiveCharacter.unmetPrerequisites`; `prereq_clauses` declared on feat (new `FieldType.prereqClauses` + `PrereqClausesFieldWidget`); built-in template version 2.4.0 → **2.5.0** (content-hash drift → lazy template-sync prompt). [[Effect-DSL-Resolution]] flow + [[Character-System]] MoC updated.
